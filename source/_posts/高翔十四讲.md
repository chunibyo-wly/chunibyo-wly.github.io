# 1. 编程基础

## cmake

1. 创建build文件夹, `cmake ..`
2. .a 静态库, 每次调用生成一个副本
3. .so 共享库, 只有一个副本

```cmake
# 声明要求的 cmake 最低版本
cmake_minimum_required(VERSION 2.8)

# 声明一个 cmake 工程
project(HelloSLAM)

# 设置编译模式
set(CMAKE_BUILD_TYPE "Debug")

# 程序1
# 语法：add_executable( 程序名 源代码文件 ）
add_executable(program1 helloSLAM.cpp)

# 程序2
add_executable(program2 useHello.cpp)

# 共享库
add_library(hello_shared SHARED libHelloSLAM.cpp)
# 将库文件链接到可执行程序上
target_link_libraries(program2 hello_shared)

```

   #include <Eigen/Core>
   #include <Eigen/Dense>
   #include <iostream>
   
   using namespace std;
   
   #define MAT_SIZE 100 //设置矩阵的规模
   #define BLOCK_SIZE 3 //设置被替换的块的规模
   
   int main()
   {
       //定义并随机初始化一个指定规模的矩阵
       Eigen::Matrix< double, MAT_SIZE, MAT_SIZE > Matrix_NN;
       Matrix_NN = Eigen::Matrix< double, MAT_SIZE, MAT_SIZE >::Random(MAT_SIZE, MAT_SIZE);
   
       //定义并初始化一个单位矩阵
       Eigen::Matrix< double, BLOCK_SIZE, BLOCK_SIZE > Matrix_Identity;
       Matrix_Identity = Eigen::Matrix< double, BLOCK_SIZE, BLOCK_SIZE >::Identity(BLOCK_SIZE, BLOCK_SIZE);
   
       Matrix_NN.block(0, 0, BLOCK_SIZE, BLOCK_SIZE) = Matrix_Identity;
   
       cout << Matrix_NN.block(0, 0, 10, 10) << endl;
   }

## 特征点

1. 特征点
   1. 可重复性（Repeatability）：相同的“区域”可以在不同的图像中被找到。
   2. 可区别性（Distinctiveness）：不同的“区域”有不同的表达。
   3. 高效率（Efficiency）：同一图像中，特征点的数量应远小于像素的数量。
   4. 本地性（Locality）：特征仅与一小片图像区域相关。
2. ORB特征
   1. FAST(Features From Accelerated Segment Test)角点提取：找出图像中的” 角点”。相较于原版的 FAST, ORB 中计算了特征 点的主方向，为后续的 BRIEF 描述子增加了旋转不变特性。
   2. BRIEF(Binary Robust Independent Elementary Features)描述子：对前一步提取出特征点的周围图像区域进行描述。
3. 特征匹配
   1. 暴力匹配（Brute-Force Matcher）
   2. 此时快速近似最 近邻（FLANN）
